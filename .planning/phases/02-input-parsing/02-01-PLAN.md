### Task 2: Create Gemini service

<files>backend/services/gemini_service.py</files>

<action>
Create `gemini_service.py` with:
1. `GeminiService` class initialized with API key from env
2. `analyze_frames(frames: list[bytes]) -> dict` method
   - Sends frames to Gemini 2.0 Flash
   - Uses structured output schema for response
   - Prompt uses Chain of Thought: observe → identify → assess → conclude
   - Uses professional Caterpillar TA1 inspection terminology
3. Output schema:
```json
{
  "chain_of_thought": {
    "observations": "What I see in the frames...",
    "component_identification": "This appears to be...",
    "condition_assessment": "The condition shows...",
    "conclusion": "Based on my analysis..."
  },
  "component": "hydraulic cylinder",
  "condition_observations": ["No visible leaks", "Surface rust on rod"],
  "preliminary_status": "MONITOR",
  "confidence": 0.85,
  "concerns": ["Surface corrosion may indicate seal wear"]
}
```
</action>

<verify>
```bash
python3 -c "
from services.gemini_service import GeminiService
svc = GeminiService()
print('GeminiService initialized with model:', svc.model)
"
```
</verify>

<done>GeminiService class instantiates and connects to Gemini API</done>
